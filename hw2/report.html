前言

這次作業要我們實作Image Stitching，把多張圖片變成一張完整的全景照片，這次作業使用的工具是OpenCV C++。

實作細節

下面是我這次程式的執行流程

Feature Detection

在這次Project中，我使用的演算法是Harris Corner Detection。這個演算法實作起來非常容易，雖然他在某些情況下出來的效果並不是那麼好，但是足以應付這次image stitching拿來作feature detection的方法。首先我會先使用3×33×3 的 Sobel Filter KernelSobel Filter Kernel 把每個pixel的Gradient X & y 算出來，這時候我可以得到每個pixel 的 gradient。拿到Gradient 之後，也可以算出矩陣M，又因為矩陣M是Quadratic function，因此對矩陣M做symmetric之後，就可以拿到兩個eigen value，並且可以算出 R=det(M)−k×trace2R=det(M)−k×trace2 (k=0.05k=0.05)。如果把每個pixel的R都算出來，就會得到一張Radiance Map(如下圖):

這時候拿到Radiance Map之後，就可以開始找Feature。令Threshold=1.1×Average Intensive SumThreshold=1.1×Average Intensive Sum，低於這個Threshold的pixel就不考慮他的local maximum。找Local maximum 則是非常值觀的去跟周圍的Pixel比較，之後就會得到拿到Pixel的features。下面是我偵測出來的feature points:

Feature Matching

抓到Feature Points之後，我們必須抓出這些feature poin 所對應到的 patch，並且去匹配不同照片上的patch，以下是我match feature points 的方法:

  1. 把之前做Harris Corner Detection 的gradient 拿來當作是descriptor(向量)，下圖是我feature point gradient 的向量示意圖

  2. 把這些Feature point 從 Gradient 的方向，全部轉到 (0,1) 的方向上。目的是為了要匹配不同照片的部分 patch 是否相同，把這些patch找出來後，每個patch就是我們的向量的元素之一，未來可以來拿做為Image matching使用。下面是我找出來不同照片的patch，可以看到藍色的箭頭都被轉到同一個方向上了:

  3. 使用KNN Match 這些patch，概念上每個配對點都是相同patch在不同照片上的位移，並且把這些位移當作是向量，下面是初步篩選的結果：

    再來就是要找屬於真正特徵點的配對，我採用的方法是多數決，去找最多相似的向量群體，並以他們當最後配對的結果。一般來說是用RANSAC來做，我在這次作業用我自己設計的演算法來篩選，其結果看起來還算不錯。以下是我找相似位移的條件:

    • 相同位移向量長度差異必須小於10%
    • 相同位移向量的角度差異小於5o5o

    從第一個位移向量開始找其他向量，當兩個向量是一樣的就把他加入同一個群組裡面，直到把所有的位移向量分類完成。分類完成後，擁有最多位移向量的群組就是我們的「多數人」。分類好的向量大概像是下面的結果:

    藍色的匹配線是經過我們篩選後所得到的匹配向量，也可以發現到少數的綠色線就會直接被淘汰，藉此抓出重要的配對點。

  4. 從這些配對結果建立相片之間的關係， 建立出鄰居之間的關係表。這可以用來查找當前相片的左右鄰居是誰，幫助相片找到在全景照片中的位置。目前的設定適當匹配的向量數超過 10 就代表這兩張照片相鄰。

Image Matching

得到相片之間的特徵點配對之後，我們可以嘗試把照片拼在一起。由於我們的相片只有兩個方向，因此對於一張相片中，只有左右兩邊會有特徵匹配到。匹配的時候去看他的位移是向左邊移動還是向右邊移動，藉此把它移動到當前相片的左邊還是右邊。那要移動多少距離，會取決於以配對到的特徵點的位移向量，直接平均這些位移項量就是其他圖片與當前圖片的位移量。以下是拿這次作業提供的 testcase 做出來matching 的結果(由於圖片過長，建議直接把圖片載下來看)：

實際成果

下面是我們自己拍的照片，在轉相機的角度的時候，可能有一點偏掉了，或者因為旋轉角度太大，wrap後的扭曲程度差異太大，alpha blending處理接縫的部份效果沒有很好。再來就是被拍攝的景物會晃動，導致兩張照片拍起來不一樣，合成出來就會有鬼影，比方說樹會隨著風吹晃動。當初拍照片的時候沒有考慮到各種細節，不過效果看起來還算是不錯的。

額外實作

由於在實作的過程中發現用Harris Corner Detection找特徵點的時候非常的慢，因此有使用OpenMP對這段code做平行化。

心得

這次作業最難的部分大概是Feature Detection 和 Feature Matching，因為不知道是自己參數沒取好還是演算法在實作上有問題，這個bug非常難找。過了這兩關之後，其實後面就實作的很快。Blending的部分是還可以繼續改進的，可惜這次沒時間implement更強大的blending演算法，所以交界處還是會有一點糊糊的。再來就是自己拍的照片要很小心有沒有動到相機的中心點，如果有不小心位移了一下，結果可能會看起來像是我們那樣，以上就是我們這次實作的Image Stitching。
